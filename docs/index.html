What it does

A complete post-quantum migration loop.

Not just a scanner — an inventory, a risk model, a worklist, and an audit trail.

01

Cryptographic discovery

A 34-pattern, language-agnostic scanner finds RSA, ECC (incl. Ed25519/X25519), DSA, Diffie-Hellman, legacy symmetric (DES/3DES/AES-128), broken hashes (MD5/SHA-1), private-key PEMs, JWT/JOSE algorithms, and X.509 certs across 25+ file types.

02

Risk scoring

A transparent 5-factor weighted model yields a 0–100 score, a priority tier, a migration-effort estimate, and a NIST PQC replacement (ML-KEM / ML-DSA / SLH-DSA). Weights are tunable per deployment.

03

Migration tracking

Mark each asset open → in progress → migrated → accepted. Status persists across re-scans, so the dashboard shows live progress and remaining effort instead of a stale report.

04

Drift detection

Every scan is diffed against the last one, flagging findings introduced and removed — catching, e.g., a PR that adds a fresh RSA usage before it merges.

05

Compliance automation

Maps findings to FISMA, CISA, FedRAMP, SOC 2, and PCI-DSS controls with pass/gap/fail status, then exports auditor-ready JSON and print-to-PDF HTML reports.

06

Standards-based exports

SARIF 2.1.0 for GitHub code-scanning, a CycloneDX 1.6 CBOM (Cryptography Bill of Materials), and filtered CSV for SIEM, spreadsheets, or tickets.

Run the full platform

Dashboard + API, one command. No external database.

# from quantumvault/
docker compose up --build
# dashboard → http://localhost:8080
# API       → http://localhost:4000

Gate PRs with the GitHub Action

Surface findings as code-scanning alerts; fail on high+.

- uses: DemigodDSK/quantumvault@v0.1.0
  with:
    path: .
    fail-on: high
    sarif-file: quantumvault.sarif
- uses: github/codeql-action/upload-sarif@v3
  with:
    sarif_file: quantumvault.sarif

Prefer the command line? The same engine ships as a quantumvault CLI with --json, --sarif, --cbom, and --csv output, plus --fail-on <severity> to gate any pipeline by exit code.
